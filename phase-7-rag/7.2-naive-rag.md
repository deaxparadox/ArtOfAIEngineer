## 1. What Is It? (Theory)

### The Minimum Viable RAG Pipeline

Naive RAG is the simplest complete implementation. It works, it ships, and it's where everyone
starts. Understanding naive RAG deeply is essential before adding complexity — because most
teams add advanced techniques before they've measured whether naive RAG is failing and why.

The naive RAG pipeline has five steps:

```
1. INGEST:     Load and parse documents
2. CHUNK:      Split documents into retrievable units
3. EMBED:      Convert chunks to vectors
4. STORE:      Insert vectors + metadata into a vector database
5. QUERY:      Embed query → retrieve top-K → generate answer
```

That's it. Steps 1–4 are the indexing pipeline (run once, update when documents change).
Step 5 is the query pipeline (run per user request).

### What "Naive" Means (and What It Doesn't)

"Naive RAG" doesn't mean bad or unusable. It means the baseline implementation without
advanced techniques. A well-implemented naive RAG with good chunking and a strong embedding
model will outperform a poorly-implemented advanced RAG.

Before reaching for advanced techniques, ask: "Have I measured whether the naive approach
is actually failing, and specifically how?"

## 2. How It Actually Works — Deep Mechanics

### 2.1 The Indexing Pipeline

```python
from pathlib import Path
from dataclasses import dataclass
from langchain.text_splitter import RecursiveCharacterTextSplitter
from sentence_transformers import SentenceTransformer
from qdrant_client import QdrantClient
from qdrant_client.models import PointStruct, VectorParams, Distance
import tiktoken
import hashlib
import json

# Configuration
EMBEDDING_MODEL = "BAAI/bge-large-en-v1.5"
VECTOR_DB_URL   = "http://localhost:6333"
COLLECTION      = "knowledge_base"
CHUNK_SIZE      = 512     # tokens
CHUNK_OVERLAP   = 64      # tokens
DIM             = 1024    # bge-large embedding dimensions

enc   = tiktoken.get_encoding("cl100k_base")
embed = SentenceTransformer(EMBEDDING_MODEL)
qdrant = QdrantClient(url=VECTOR_DB_URL)

@dataclass
class DocumentChunk:
    chunk_id:    str
    doc_id:      str
    text:        str
    token_count: int
    chunk_index: int
    source:      str
    metadata:    dict

def setup_collection():
    """Create or recreate the vector collection."""
    qdrant.recreate_collection(
        collection_name=COLLECTION,
        vectors_config=VectorParams(size=DIM, distance=Distance.COSINE)
    )

def load_document(path: str) -> str:
    """Load text from a file. In production: use appropriate parsers for PDF, HTML, etc."""
    suffix = Path(path).suffix.lower()
    if suffix == ".txt":
        return Path(path).read_text(encoding="utf-8")
    elif suffix == ".md":
        return Path(path).read_text(encoding="utf-8")
    else:
        raise ValueError(f"Unsupported format: {suffix}")

def chunk_document(text: str, doc_id: str, source: str, metadata: dict = {}) -> list[DocumentChunk]:
    """Split document into overlapping chunks with stable IDs."""
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=CHUNK_SIZE,
        chunk_overlap=CHUNK_OVERLAP,
        length_function=lambda t: len(enc.encode(t)),
        separators=["\n\n", "\n", ". ", " ", ""],
    )
    raw_chunks = splitter.split_text(text)
    
    chunks = []
    for i, text_chunk in enumerate(raw_chunks):
        # Deterministic chunk ID — enables idempotent re-indexing
        chunk_id = hashlib.md5(f"{doc_id}:{i}:{text_chunk[:50]}".encode()).hexdigest()
        chunks.append(DocumentChunk(
            chunk_id    = chunk_id,
            doc_id      = doc_id,
            text        = text_chunk,
            token_count = len(enc.encode(text_chunk)),
            chunk_index = i,
            source      = source,
            metadata    = {**metadata, "chunk_index": i, "total_chunks": len(raw_chunks)}
        ))
    return chunks

def index_documents(paths: list[str]) -> int:
    """Load, chunk, embed, and store documents. Returns total chunks indexed."""
    all_chunks: list[DocumentChunk] = []
    
    for path in paths:
        doc_id = hashlib.md5(path.encode()).hexdigest()[:12]
        text   = load_document(path)
        chunks = chunk_document(text, doc_id, source=path, metadata={"file": path})
        all_chunks.extend(chunks)
        print(f"  {path}: {len(chunks)} chunks")
    
    print(f"\nEmbedding {len(all_chunks)} chunks...")
    texts = [c.text for c in all_chunks]
    embeddings = embed.encode(texts, normalize_embeddings=True, batch_size=32, show_progress_bar=True)
    
    print("Storing in vector database...")
    points = [
        PointStruct(
            id      = abs(hash(c.chunk_id)) % (2**53),   # Qdrant needs integer or UUID IDs
            vector  = embeddings[i].tolist(),
            payload = {
                "text":        c.text,
                "doc_id":      c.doc_id,
                "chunk_id":    c.chunk_id,
                "source":      c.source,
                "chunk_index": c.chunk_index,
                **c.metadata,
            }
        )
        for i, c in enumerate(all_chunks)
    ]
    qdrant.upsert(collection_name=COLLECTION, points=points)
    
    print(f"Indexed {len(all_chunks)} chunks from {len(paths)} documents.")
    return len(all_chunks)
```

### 2.2 The Query Pipeline

```python
import anthropic

llm = anthropic.Anthropic()

def retrieve(query: str, top_k: int = 5) -> list[dict]:
    """Embed query and retrieve top-K most relevant chunks."""
    q_vec = embed.encode([query], normalize_embeddings=True)[0]
    
    results = qdrant.search(
        collection_name=COLLECTION,
        query_vector=q_vec.tolist(),
        limit=top_k,
        with_payload=True
    )
    
    return [
        {
            "text":   r.payload["text"],
            "source": r.payload["source"],
            "score":  r.score,
            "chunk_index": r.payload.get("chunk_index", 0)
        }
        for r in results
    ]

def build_context(chunks: list[dict]) -> str:
    """Format retrieved chunks into a context string for the LLM."""
    parts = []
    for i, chunk in enumerate(chunks, start=1):
        source = chunk["source"].split("/")[-1]   # just the filename
        parts.append(f"[Source {i} — {source}]\n{chunk['text']}")
    return "\n\n---\n\n".join(parts)

def generate_answer(query: str, context: str) -> str:
    """Generate an answer grounded in the retrieved context."""
    response = llm.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system="""You are a helpful assistant that answers questions based on provided documents.

Rules:
1. Use ONLY information from the provided sources. Do not add outside knowledge.
2. If the answer is not in the sources, say: "The provided documents don't contain this information."
3. Cite sources using [Source N] notation for each factual claim.
4. Be concise. Don't repeat information from the sources verbatim when paraphrasing is clearer.""",
        messages=[{
            "role": "user",
            "content": f"""Sources:
{context}

Question: {query}"""
        }]
    )
    return response.content[0].text

def rag_query(question: str, top_k: int = 5, verbose: bool = False) -> dict:
    """Complete naive RAG pipeline: retrieve → generate → return."""
    # Step 1: Retrieve
    chunks = retrieve(question, top_k=top_k)
    
    if verbose:
        print(f"\nRetrieved {len(chunks)} chunks:")
        for c in chunks:
            print(f"  [{c['score']:.3f}] {c['text'][:80]}...")
    
    # Step 2: Build context
    context = build_context(chunks)
    
    # Step 3: Generate
    answer = generate_answer(question, context)
    
    return {
        "question":  question,
        "answer":    answer,
        "chunks":    chunks,
        "context":   context,
        "top_k":     top_k,
    }
```

### 2.3 Complete End-to-End Example

```python
# Full usage example

# 1. Setup
setup_collection()

# 2. Index some documents
# (In a real system, these would be your actual knowledge base files)
import tempfile, os

sample_docs = {
    "return_policy.txt": """
# Return Policy

Our return policy allows returns within 30 days of purchase.
Products must be in original, unopened condition. 
Sale items are final sale and cannot be returned.
Refunds are processed within 5-7 business days.
For online orders, use the prepaid return label from your account.
    """,
    "shipping_info.txt": """
# Shipping Information

Standard shipping takes 5-7 business days.
Express shipping (2-day) is available for an extra $15.
Free standard shipping on orders over $50.
We ship to all 50 US states and Canada.
International shipping is available for select countries.
    """
}

tmp_paths = []
with tempfile.TemporaryDirectory() as tmpdir:
    for filename, content in sample_docs.items():
        path = os.path.join(tmpdir, filename)
        with open(path, "w") as f:
            f.write(content)
        tmp_paths.append(path)
    
    index_documents(tmp_paths)

# 3. Query
result = rag_query("Can I return a sale item if it's unopened?", top_k=3, verbose=True)
print("\n" + "="*60)
print("Answer:")
print(result["answer"])
```

### 2.4 System Prompt Engineering for RAG

The system prompt for a RAG pipeline needs to:
1. Establish the grounding contract (use only provided sources)
2. Define out-of-scope behavior (what to say when the answer isn't in the docs)
3. Specify citation format
4. Handle contradictions in the sources

```python
RAG_SYSTEM_PROMPT = """You are a knowledgeable assistant with access to a curated document library.

## Your Rules

SOURCES ONLY: Answer using only information from the provided [Source N] blocks.
Do not use your general knowledge, even if you believe it to be correct.

NOT IN SOURCES: If the question cannot be answered from the sources, respond with:
"The provided documents don't contain information about [topic]. Please consult [appropriate resource]."

CITATIONS: After every factual claim, add [Source N] where N is the source number.
Multiple sources: [Source 1][Source 3].

CONTRADICTIONS: If sources contradict each other, note it:
"Source 1 states X, while Source 3 states Y. Please verify with [appropriate team]."

UNCERTAINTY: If you're unsure whether a source supports a claim, say so explicitly.
Never guess or extrapolate beyond what the sources state."""
```

## 3. Real-World Context

### What Naive RAG Gets Right

When properly implemented, naive RAG handles the majority of real-world RAG use cases adequately:
- Single-hop questions (the answer is in one chunk)
- Questions with clear, specific answers
- Q&A over well-structured, clean documents
- Applications where 80–85% answer accuracy is acceptable

### What Naive RAG Gets Wrong (That Advanced RAG Fixes)

| Failure | Root Cause | Advanced Fix (Section 7.3) |
|---|---|---|
| "Wrong answer" from wrong chunk | Retrieval miss | Reranking, hybrid search |
| Partial answer | Query covers multiple chunks | Query expansion, multi-query |
| Good chunks but LLM ignores them | Phrasing asymmetry | HyDE, better prompting |
| Hallucination despite context | LLM overrides context | Citation forcing, faithfulness check |
| Short/ambiguous query fails | Semantic gap | Query expansion |
| Long document loses context | Chunking splits answers | Parent-child retrieval |

### Production Latency Budget

```
Naive RAG per request:
  Embed query:          10–30ms  (cached with prompt caching for repeated queries)
  Vector search:         2–10ms  (depends on index size)
  LLM generation:     300–2000ms (dominant cost)
  ─────────────────────────────
  Total:              312–2040ms

Optimization targets:
  - Use a fast/cheap model (Claude Haiku, GPT-4o-mini) for simple questions
  - Cache common query embeddings
  - Use vector DB with fast query latency (Qdrant, Pinecone)
```

def load_pdf(path: str) -> str:
    reader = PdfReader(path)

import requests
from bs4 import BeautifulSoup

def load_webpage(url: str) -> str:
    html = requests.get(url, timeout=10).text
    soup = BeautifulSoup(html, "html.parser")
    # Remove nav, footer, scripts

## 6. Trade-offs

### Strict Citation vs Fluent Generation

**Strict grounding** (cite every claim):
- Maximum hallucination prevention
- Slightly stilted responses
- Clear audit trail

**Flexible generation** (use context, but write naturally):
- More natural responses
- Risk of the model paraphrasing loosely or inferring beyond the sources
- Harder to audit

For high-stakes applications (legal, medical, financial): strict grounding.
For conversational applications: flexible with citation on demand.

## 7. When To Use vs. When To Avoid

### Naive RAG
✅ Start here: Every RAG system should start with naive RAG, measure quality, then upgrade
✅ Use when: Single-hop Q&A over structured documents, quality target is achievable
❌ Upgrade to advanced when: Recall@5 < 0.75 on your eval set, or users report missing answers

⚠️ The most common mistake: Adding advanced RAG techniques before measuring whether naive RAG
is failing, and specifically why. If your retrieval recall is already 90%, adding query expansion
won't help — the problem is elsewhere.

## 8. Senior AI Engineer Perspective

### The Importance of the Citation Pattern

The single most impactful thing in naive RAG is enforcing source citations in the system prompt.
Without citations, you can't tell whether the LLM is using the retrieved context or falling back
on training memory. With citations, every factual claim points to a source — and you can audit.

Build citation checking into your eval suite: parse citations from the answer, verify they exist
in the retrieved chunks, verify the chunk actually supports the claim. This is your best signal
for faithfulness quality.

### The Chunk as Context Unit

The chunk you retrieve is not just used as an information source — it's the entire context the
model has for that piece of knowledge. If the chunk is:
- Too short: missing context that explains the text
- Too long: diluted by surrounding irrelevant content
- Split mid-sentence: semantically broken
- Cut before the key information: the model sees setup but not the answer

Getting chunking right is the highest-leverage investment in naive RAG. Spend 30% of your
retrieval engineering time here before moving to other optimizations.

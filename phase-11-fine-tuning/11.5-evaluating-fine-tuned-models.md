## 1. What Is It? (Theory)

### The Evaluation Gap

The most common fine-tuning mistake that doesn't involve data: shipping a fine-tuned model
without rigorously comparing it to the baseline.

Teams check:
- "Training loss went down" ✓ (expected; not sufficient)
- "Tried a few examples, looks better" ✓ (qualitative; not sufficient)
- "Our eval benchmark is better" — often this benchmark was too easy or too narrow

They miss:
- Is it better than the baseline on the ACTUAL task distribution?
- Did it get better on the target task while getting worse on related tasks?
- Are there new failure modes that didn't exist in the base model?

Fine-tuning evaluation requires all the discipline from Phase 9, plus fine-tuning-specific
concerns like catastrophic forgetting and overfitting.

### What You Must Compare

```
Comparison 1: Fine-tuned vs Base model (same prompting)
  Question: Did fine-tuning help at all?

Comparison 2: Fine-tuned vs Prompted base (optimized prompt)
  Question: Does fine-tuning beat the best we can do with prompting?
  (This is the real comparison most teams skip)

Comparison 3: Fine-tuned vs Previous fine-tuned version
  Question: Is the new version actually better? (Regression testing)
```

## 2. How It Actually Works — Deep Mechanics

### 2.1 The Evaluation Suite Structure

```python
from dataclasses import dataclass
import anthropic
import json
import numpy as np

client = anthropic.Anthropic()

@dataclass
class ModelVersion:
    name: str
    model_id: str
    system_prompt: str

# Define the models to compare
VERSIONS = {
    "base_model": ModelVersion(
        name="Base Claude Haiku",
        model_id="claude-haiku-4-5-20251001",
        system_prompt="You are a customer support assistant."
    ),
    "base_prompted": ModelVersion(
        name="Base + Optimized Prompt",
        model_id="claude-haiku-4-5-20251001",
        system_prompt="""You are a customer support assistant. 
Always respond in this exact JSON format:
{
  "answer": "...",
  "confidence": "high|medium|low",
  "action_needed": true|false
}"""
    ),
    "fine_tuned": ModelVersion(
        name="Fine-Tuned v1",
        model_id="ft:gpt-4o-mini:company:support-v1:abc123",   # replace with actual ID
        system_prompt="You are a customer support assistant."
    ),
}

def compare_models(
    eval_cases: list[dict],
    models: dict[str, ModelVersion],
) -> dict:
    """
    Run all model versions against all eval cases.
    Returns per-model scores for comparison.
    """
    results = {name: {"scores": [], "outputs": []} for name in models}
    
    for case in eval_cases:
        for model_name, version in models.items():
            response = client.messages.create(
                model=version.model_id,
                max_tokens=256,
                system=version.system_prompt,
                messages=[{"role": "user", "content": case["input"]}],
                temperature=0
            )
            output = response.content[0].text
            
            # Score deterministically where possible
            score = score_response(case, output)
            
            results[model_name]["scores"].append(score)
            results[model_name]["outputs"].append({
                "input":    case["input"],
                "output":   output,
                "expected": case.get("expected", ""),
                "score":    score
            })
    
    # Aggregate
    summary = {}
    for name, data in results.items():
        scores = data["scores"]
        summary[name] = {
            "mean_score":   round(np.mean(scores), 3),
            "pass_rate":    round(sum(s >= 0.7 for s in scores) / len(scores), 3),
            "p10_score":    round(np.percentile(scores, 10), 3),
            "n_eval_cases": len(scores),
        }
    
    return {"per_model": summary, "per_case": results}

def score_response(case: dict, output: str) -> float:
    """Score a response — combine deterministic and semantic checks."""
    score = 0.0
    weights = 0.0
    
    # Check: is expected content present?
    if "expected_keywords" in case:
        matches = sum(kw.lower() in output.lower() for kw in case["expected_keywords"])
        score += matches / len(case["expected_keywords"]) * 0.5
        weights += 0.5
    
    # Check: is output format correct?
    if "expected_format" in case:
        if case["expected_format"] == "json":
            try:
                json.loads(output)
                score += 0.3
            except json.JSONDecodeError:
                pass
        weights += 0.3
    
    # Check: is output length reasonable?
    words = len(output.split())
    if 5 <= words <= 200:
        score += 0.2
    weights += 0.2
    
    return score / weights if weights > 0 else 0.0
```

### 2.2 Catastrophic Forgetting Check

Fine-tuning on a narrow task can degrade general capabilities. Always test this:

```python
def check_catastrophic_forgetting(
    fine_tuned_model_id: str,
    base_model_id: str,
    general_capability_cases: list[dict],   # cases testing general knowledge
) -> dict:
    """
    Check if fine-tuning degraded general capabilities.
    A fine-tuned model should NOT be significantly worse at:
    - General question answering
    - Instruction following
    - Reasoning tasks
    """
    base_scores = []
    ft_scores   = []
    
    for case in general_capability_cases:
        for model_id, scores_list in [(base_model_id, base_scores), (fine_tuned_model_id, ft_scores)]:
            response = client.messages.create(
                model=model_id, max_tokens=256,
                messages=[{"role": "user", "content": case["input"]}],
                temperature=0
            )
            score = score_response(case, response.content[0].text)
            scores_list.append(score)
    
    base_mean = np.mean(base_scores)
    ft_mean   = np.mean(ft_scores)
    delta     = (ft_mean - base_mean) / base_mean
    
    return {
        "base_mean":          round(base_mean, 3),
        "fine_tuned_mean":    round(ft_mean, 3),
        "relative_change":    f"{delta:+.1%}",
        "forgetting_detected": delta < -0.05,  # >5% degradation
        "verdict": "⚠️  FORGETTING DETECTED" if delta < -0.05 else "✓ No significant forgetting",
    }

# General capability test cases (always include in your eval suite)
GENERAL_CASES = [
    {"input": "What is 15% of 200?",           "expected_keywords": ["30"]},
    {"input": "Translate 'hello' to Spanish.",  "expected_keywords": ["hola"]},
    {"input": "What is the capital of Japan?",  "expected_keywords": ["Tokyo"]},
    {"input": "Summarize: The quick brown fox...", "expected_keywords": ["fox", "dog"]},
    {"input": "Write a Python function to add two numbers.",
     "expected_keywords": ["def", "return", "+"]},
]
```

### 2.3 Training Loss Monitoring

```python
def analyze_training_curves(job_events: list) -> dict:
    """
    Analyze training dynamics to diagnose training quality.
    Works for OpenAI fine-tuning event logs.
    """
    train_losses = []
    val_losses   = []
    
    for event in job_events:
        msg = event.get("message", "")
        if "train_loss=" in msg:
            try:
                train_loss = float(msg.split("train_loss=")[1].split(",")[0])
                train_losses.append(train_loss)
            except (ValueError, IndexError):
                pass
        if "val_loss=" in msg:
            try:
                val_loss = float(msg.split("val_loss=")[1].split(",")[0])
                val_losses.append(val_loss)
            except (ValueError, IndexError):
                pass
    
    if not train_losses:
        return {"error": "No loss data found"}
    
    # Diagnose training health
    final_train = train_losses[-1]
    final_val   = val_losses[-1] if val_losses else None
    
    issues = []
    
    # Check: did training converge?
    if len(train_losses) > 5:
        early_avg = np.mean(train_losses[:5])
        late_avg  = np.mean(train_losses[-5:])
        if late_avg > early_avg * 0.9:
            issues.append("Training loss barely decreased — model may not have learned")
    
    # Check: overfitting
    if final_val and final_val > final_train * 1.2:
        issues.append(f"Validation loss ({final_val:.3f}) >> training loss ({final_train:.3f}) — overfitting")
    
    # Check: loss divergence
    if final_train > train_losses[0] * 1.5:
        issues.append("Training loss increased over time — learning rate may be too high")
    
    return {
        "initial_train_loss":  round(train_losses[0], 4) if train_losses else None,
        "final_train_loss":    round(final_train, 4),
        "final_val_loss":      round(final_val, 4) if final_val else None,
        "n_steps":             len(train_losses),
        "converged":           len(issues) == 0,
        "issues":              issues,
    }
```

### 2.4 Complete Evaluation Report

```python
def full_evaluation_report(
    base_model_id: str,
    fine_tuned_model_id: str,
    task_eval_cases: list[dict],
    general_eval_cases: list[dict],
    job_events: list | None = None,
) -> dict:
    """
    Complete evaluation comparing fine-tuned vs base model.
    Returns a structured report with deployment recommendation.
    """
    print("Running task-specific evaluation...")
    task_comparison = compare_models(
        task_eval_cases,
        {
            "base":         ModelVersion("Base", base_model_id, "Be helpful."),
            "fine_tuned":   ModelVersion("FT v1", fine_tuned_model_id, "Be helpful."),
        }
    )
    
    print("Checking for catastrophic forgetting...")
    forgetting_check = check_catastrophic_forgetting(
        fine_tuned_model_id, base_model_id, general_eval_cases
    )
    
    print("Analyzing training curves...")
    training_health = analyze_training_curves(job_events) if job_events else {"skipped": True}
    
    # Determine deployment recommendation
    task_delta = (
        task_comparison["per_model"]["fine_tuned"]["mean_score"] -
        task_comparison["per_model"]["base"]["mean_score"]
    )
    
    should_deploy = (
        task_delta >= 0.05 and                            # at least 5% improvement
        not forgetting_check.get("forgetting_detected")  # no significant forgetting
    )
    
    recommendation = "DEPLOY" if should_deploy else "DO NOT DEPLOY"
    if task_delta < 0:
        recommendation += " — fine-tuning DEGRADED task performance"
    elif task_delta < 0.05:
        recommendation += " — improvement too small to justify deployment cost"
    
    return {
        "recommendation":    recommendation,
        "task_improvement":  f"{task_delta:+.1%}",
        "task_scores":       task_comparison["per_model"],
        "forgetting":        forgetting_check,
        "training_health":   training_health,
    }
```

## 3. Real-World Context

### The Four Outcomes of Fine-Tuning Evaluation

```
Outcome 1: CLEAR WIN
  Task score +10%+, no forgetting, training converged cleanly.
  Deploy. This is what you hoped for.

Outcome 2: MARGINAL IMPROVEMENT
  Task score +2-5%. Is the quality gain worth the serving cost?
  For 10% of users, the improvement might matter.
  Decision depends on your product requirements.

Outcome 3: NO IMPROVEMENT
  Training converged but task score same as base.
  Usually: the problem is data quality, not training.
  Review and improve your training examples.

Outcome 4: REGRESSION
  Task score went down, or forgetting detected.
  Never deploy. The fine-tuning damaged the model.
  Most common cause: too few epochs, bad data, or wrong task framing.
```

## 4. Senior AI Engineer Perspective

### The Prompted Baseline Is Your Real Baseline

The most common evaluation mistake: comparing fine-tuned vs base model with the same naive
prompt. This stacks the deck for fine-tuning.

The honest comparison is: **fine-tuned with simple prompt vs base model with the best prompt you can write**.

If a well-crafted few-shot prompt on the base model matches the fine-tuned model's quality,
you didn't need to fine-tune. You needed to prompt better.

Only when fine-tuning beats the *best* prompted baseline do you have clear justification.

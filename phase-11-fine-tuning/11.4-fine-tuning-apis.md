## 1. What Is It? (Theory)

### Fine-Tuning Without a GPU

If you don't have GPU infrastructure, or don't want to manage training yourself, managed
fine-tuning APIs let you upload your data, configure training parameters, and get back a
fine-tuned model — all via API calls.

**The trade-off:** You give up flexibility and control in exchange for zero infrastructure
overhead. You can't inspect training dynamics as closely, you don't control the hardware,
and the resulting model is typically served through their API (can't export weights).

**2026 landscape:**

| Provider | Models | Status | Notes |
|---|---|---|---|
| **OpenAI** | GPT-4o-mini, GPT-4o, GPT-3.5 Turbo | Active (check for new-user access) | Most mature API |
| **Together AI** | LLaMA 4, Mistral, Qwen, etc. | Active | Best for open models |
| **Fireworks AI** | LLaMA, Mistral, custom | Active | Fast serving post-fine-tune |
| **Vertex AI** | Gemini, LLaMA | Active | GCP-native |
| **Anthropic** | Claude 3 Haiku | Limited access | Contact sales |
| **AWS Bedrock** | Various | Active | AWS-native |

## 2. How It Actually Works

### 2.1 OpenAI Fine-Tuning API

```python
from openai import OpenAI
import json
import time

client = OpenAI()

# Step 1: Prepare data in JSONL format
training_data = [
    {
        "messages": [
            {"role": "system",    "content": "You are a customer support assistant. Be concise."},
            {"role": "user",      "content": "What is your return policy?"},
            {"role": "assistant", "content": "Returns accepted within 30 days with receipt."}
        ]
    },
    {
        "messages": [
            {"role": "system",    "content": "You are a customer support assistant. Be concise."},
            {"role": "user",      "content": "Can I return a sale item?"},
            {"role": "assistant", "content": "Sale items are final — no returns or exchanges."}
        ]
    },
    # Minimum: 10 examples. Recommended: 50-100+ for meaningful improvement.
    # OpenAI will split ~20% for validation automatically.
]

# Save to file
with open("training_data.jsonl", "w") as f:
    for ex in training_data:
        f.write(json.dumps(ex) + "\n")

# Step 2: Upload training file
print("Uploading training data...")
with open("training_data.jsonl", "rb") as f:
    upload_response = client.files.create(file=f, purpose="fine-tune")

file_id = upload_response.id
print(f"Uploaded file: {file_id}")

# Step 3: Create fine-tuning job
print("Creating fine-tuning job...")
job = client.fine_tuning.jobs.create(
    training_file = file_id,
    model         = "gpt-4o-mini",   # cheapest and fastest to fine-tune
    hyperparameters = {
        "n_epochs": 3,                 # "auto" lets OpenAI decide
        "batch_size": "auto",
        "learning_rate_multiplier": 1.0,  # scale factor on default LR
    },
    suffix="support-v1",              # model ID will include this suffix
)

job_id = job.id
print(f"Job created: {job_id}")

# Step 4: Poll for completion
print("Waiting for fine-tuning to complete...")
while True:
    job = client.fine_tuning.jobs.retrieve(job_id)
    print(f"Status: {job.status}")
    
    if job.status == "succeeded":
        model_id = job.fine_tuned_model
        print(f"\nFine-tuning complete! Model: {model_id}")
        break
    elif job.status == "failed":
        print(f"Fine-tuning failed: {job.error}")
        break
    
    time.sleep(60)

# Step 5: Use the fine-tuned model
response = client.chat.completions.create(
    model=model_id,   # e.g., "ft:gpt-4o-mini:company:support-v1:abc123"
    messages=[
        {"role": "system", "content": "You are a customer support assistant. Be concise."},
        {"role": "user",   "content": "Do you accept returns?"}
    ]
)
print(response.choices[0].message.content)
```

### 2.3 Together AI Fine-Tuning (Open Models)

Together AI is the recommended platform for fine-tuning open-weight models via API.
You get the model weights back and can self-host or serve through Together.

```python
# pip install together

import together
import json
import os

together.api_key = os.environ["TOGETHER_API_KEY"]

# Together expects JSONL with {"text": "<full formatted conversation>"}
# OR the same messages format as OpenAI

def prepare_together_format(examples: list[dict]) -> str:
    """Convert messages format to Together AI format."""
    # Together supports OpenAI-compatible format for most models
    together_examples = []
    for ex in examples:
        together_examples.append({"messages": ex["messages"]})
    
    formatted = "\n".join(json.dumps(ex) for ex in together_examples)
    return formatted

# Upload file
file_resp = together.Files.upload(file="training_data.jsonl")
file_id   = file_resp["id"]

# Create fine-tuning job
job = together.FineTuning.create(
    training_file   = file_id,
    model           = "meta-llama/Llama-3-8b-Instruct",
    n_epochs        = 3,
    n_checkpoints   = 1,
    batch_size      = 4,
    learning_rate   = 1e-5,      # lower than OpenAI default
    suffix          = "support-v1",
    wandb_api_key   = os.environ.get("WANDB_API_KEY"),  # optional W&B logging
)

job_id = job["id"]
print(f"Job started: {job_id}")

# Poll
while True:
    status = together.FineTuning.retrieve(job_id)
    print(f"Status: {status['status']}")
    if status["status"] in ("completed", "failed"):
        break
    time.sleep(60)

# The fine-tuned model is now available through Together's inference API
# AND you get the weights back:
model_output_path = status.get("output_name")
print(f"Fine-tuned model: {model_output_path}")

# Inference
resp = together.Complete.create(
    prompt  = "<|im_start|>user\nWhat is your return policy?<|im_end|>\n<|im_start|>assistant\n",
    model   = model_output_path,
    max_tokens = 100,
)
print(resp["choices"][0]["text"])
```

### 2.4 Cost Comparison for API Fine-Tuning

```python
# Approximate costs (verify at provider's current pricing)
FINETUNING_COSTS = {
    "openai_gpt4o_mini": {
        "training": "$3.00/1M tokens",
        "inference_input":  "$0.30/1M tokens",  # 2× base price
        "inference_output": "$1.20/1M tokens",
        "note": "Fine-tuned models cost 2-4× more than base"
    },
    "openai_gpt4o": {
        "training": "$25.00/1M tokens",
        "inference_input":  "$3.75/1M tokens",
        "inference_output": "$15.00/1M tokens",
    },
    "together_llama3_8b": {
        "training": "$3.00/1M tokens",
        "inference_input":  "$0.20/1M tokens",
        "inference_output": "$0.30/1M tokens",
        "note": "Open model — can also self-host after fine-tuning"
    },
}

def estimate_finetuning_cost(
    n_examples: int,
    avg_tokens_per_example: int = 500,
    n_epochs: int = 3,
    provider: str = "openai_gpt4o_mini"
) -> dict:
    """Estimate total fine-tuning cost."""
    total_training_tokens = n_examples * avg_tokens_per_example * n_epochs
    
    # OpenAI pricing (check current rates)
    if "openai" in provider:
        training_cost = total_training_tokens / 1_000_000 * 3.00  # gpt-4o-mini
    else:
        training_cost = total_training_tokens / 1_000_000 * 3.00  # together
    
    return {
        "training_tokens":  total_training_tokens,
        "training_cost":    f"${training_cost:.2f}",
        "provider":         provider,
    }

# Example
cost = estimate_finetuning_cost(n_examples=1000, avg_tokens_per_example=400, n_epochs=3)
print(f"Estimated training cost: {cost['training_cost']}")
# ~$3.60 for 1,000 examples at 400 tokens each, 3 epochs on gpt-4o-mini
```

## 3. Providers, Tools & Ecosystem

### Managed Fine-Tuning Comparison (2026)

| Provider | Models | Export Weights | Price/1M Train Tokens | When to Choose |
|---|---|---|---|---|
| **OpenAI** | GPT-4o-mini, GPT-4o | No | $3–$25 | Need GPT quality; simplest API |
| **Together AI** | LLaMA 4, Mistral, Qwen | Yes | $3–$10 | Want open model + weights back |
| **Fireworks AI** | LLaMA 4, custom | Yes | Similar | Fastest serving after fine-tune |
| **Vertex AI** | Gemini, LLaMA | Partial | GCP pricing | Already on GCP |
| **Anthropic** | Claude 3 Haiku | No | Enterprise only | Contact sales |
| **AWS Bedrock** | LLaMA, others | Partial | AWS pricing | Already on AWS |

**Key question:** Can you export the weights? If yes (Together, Fireworks), you can self-host
later for cost savings. If no (OpenAI), you're locked into their serving pricing forever.

## 4. Trade-offs

### API Fine-Tuning vs Self-Hosted (LoRA/QLoRA)

| Aspect | API Fine-Tuning | Self-Hosted (LoRA/QLoRA) |
|---|---|---|
| Infrastructure | Zero | Requires GPU |
| Control | Limited | Full |
| Training visibility | Basic logs | Full W&B dashboard |
| Cost | Pay per token | Fixed GPU cost |
| Weight ownership | Provider keeps | You keep |
| Time to start | Minutes | Hours (setup) |
| Best model access | Frontier models (GPT-4o) | Open models only |

## 5. Senior AI Engineer Perspective

### Start with API Fine-Tuning, Migrate if Needed

For teams without GPU infrastructure, API fine-tuning is the fastest path to a fine-tuned
model. Start here. If costs become prohibitive or you need more control, migrate to self-hosted.

The migration path:
1. Fine-tune on Together AI (they return weights)
2. Receive LoRA adapters or merged model weights
3. Host with vLLM on your own GPU infrastructure
4. Serve at a fraction of the per-token API cost

This migration is straightforward because Together AI uses standard formats compatible with
Hugging Face. OpenAI fine-tuning is a dead end for self-hosting — no weights returned.

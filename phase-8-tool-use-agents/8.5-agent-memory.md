## 1. What Is It? (Theory)

### The Memory Problem

An LLM has no memory between API calls. Every request starts fresh. Without memory architecture:

```
Session 1:  User: "My name is Alice and I'm building a Python API."
            Agent: "Got it, Alice! What would you like help with?"

Session 2 (next day):
            User: "Can you review what we built yesterday?"
            Agent: "I don't have any information about what you built yesterday."
```

For long-running assistants, complex agents, or personalized experiences, this statelessness
is a fundamental limitation. Memory systems solve it.

### The Four Memory Types

Modern agent memory research distinguishes four types, each serving a different purpose:

```
SHORT-TERM / WORKING MEMORY:
  What: The conversation context in the current session
  Storage: The LLM's context window (already exists!)
  Limit: Context window size (200K–2M tokens)
  Duration: Single session

LONG-TERM / SEMANTIC MEMORY:
  What: Facts about the user, world, domain knowledge
  Storage: Database (key-value, relational, vector)
  Limit: Unlimited (but retrieval needed to use it)
  Duration: Persistent across sessions

EPISODIC MEMORY:
  What: Records of past interactions, what happened when
  Storage: Vector DB (for semantic retrieval) + structured DB (for lookup)
  Limit: Grows over time
  Duration: Persistent; selectively retrieved

PROCEDURAL MEMORY:
  What: How to do things; learned strategies; skills
  Storage: Fine-tuning or tool definitions
  Limit: Model weights or tool schema
  Duration: Permanent (until model is updated)
```

## 2. How It Actually Works — Deep Mechanics

### 2.1 Short-Term Memory: Context Window Management

The simplest memory is just the conversation history in context. The challenge is managing
it as it grows:

```python
from collections import deque
import tiktoken

enc = tiktoken.get_encoding("cl100k_base")

class ConversationMemory:
    """
    Sliding window conversation memory.
    Keeps recent context while staying within token budget.
    """
    
    def __init__(
        self,
        system_prompt: str,
        max_context_tokens: int = 100_000,
        min_turns_to_keep: int = 6,    # always keep at least 3 exchanges
    ):
        self.system_prompt       = system_prompt
        self.max_context_tokens  = max_context_tokens
        self.min_turns_to_keep   = min_turns_to_keep
        self.messages: list[dict] = []
        self._system_tokens      = len(enc.encode(system_prompt))
    
    def add(self, role: str, content: str):
        self.messages.append({"role": role, "content": content})
    
    def get_messages(self) -> list[dict]:
        """Return messages that fit within the token budget."""
        # Reserve tokens for system prompt + LLM output
        available = self.max_context_tokens - self._system_tokens - 4096
        
        # Count from the end, keeping as many recent messages as possible
        included = []
        tokens_used = 0
        
        for msg in reversed(self.messages):
            msg_tokens = len(enc.encode(msg["content"])) + 10  # + framing overhead
            if tokens_used + msg_tokens > available and len(included) >= self.min_turns_to_keep:
                break
            included.insert(0, msg)
            tokens_used += msg_tokens
        
        return included
    
    def summarize_old_turns(self, summarizer_fn) -> None:
        """
        When context grows too large, summarize old turns into a single message.
        """
        if len(self.messages) <= self.min_turns_to_keep:
            return
        
        to_summarize = self.messages[:-self.min_turns_to_keep]
        recent       = self.messages[-self.min_turns_to_keep:]
        
        summary_text = summarizer_fn(to_summarize)
        
        # Replace old messages with summary
        self.messages = [
            {"role": "user", "content": f"[Earlier conversation summary]\n{summary_text}"},
            {"role": "assistant", "content": "Understood, I have the context from our earlier conversation."}
        ] + recent
```

### 2.2 Long-Term Semantic Memory: Persistent Facts

Store and retrieve facts about users, preferences, and domain knowledge across sessions:

```python
import json
from datetime import datetime
from qdrant_client import QdrantClient
from qdrant_client.models import PointStruct, VectorParams, Distance
from sentence_transformers import SentenceTransformer

embed = SentenceTransformer("all-MiniLM-L6-v2")
qdrant = QdrantClient(url="http://localhost:6333")

class SemanticMemoryStore:
    """
    Long-term semantic memory: facts stored as embeddings, retrieved by semantic similarity.
    """
    
    def __init__(self, collection: str = "agent_memory", user_id: str = "default"):
        self.collection = collection
        self.user_id    = user_id
        
        try:
            qdrant.get_collection(collection)
        except Exception:
            qdrant.recreate_collection(
                collection_name=collection,
                vectors_config=VectorParams(size=384, distance=Distance.COSINE)
            )
    
    def remember(self, fact: str, category: str = "general", importance: float = 0.5):
        """Store a fact in long-term memory."""
        vec = embed.encode([fact], normalize_embeddings=True)[0]
        point = PointStruct(
            id=abs(hash(f"{self.user_id}:{fact}")) % (2**53),
            vector=vec.tolist(),
            payload={
                "fact":       fact,
                "user_id":    self.user_id,
                "category":   category,
                "importance": importance,
                "created_at": datetime.utcnow().isoformat(),
            }
        )
        qdrant.upsert(collection_name=self.collection, points=[point])
    
    def recall(self, query: str, k: int = 5, min_score: float = 0.5) -> list[dict]:
        """Retrieve relevant memories for the current context."""
        q_vec = embed.encode([query], normalize_embeddings=True)[0]
        results = qdrant.search(
            collection_name=self.collection,
            query_vector=q_vec.tolist(),
            limit=k,
            with_payload=True,
        )
        return [
            {"fact": r.payload["fact"], "score": r.score, "category": r.payload["category"]}
            for r in results if r.score >= min_score
        ]
    
    def forget(self, fact_pattern: str):
        """Remove facts matching a pattern."""
        results = self.recall(fact_pattern, k=10)
        ids = [abs(hash(f"{self.user_id}:{r['fact']}")) % (2**53) for r in results]
        if ids:
            qdrant.delete(collection_name=self.collection, points_selector=ids)


# Usage in a personalized assistant
memory = SemanticMemoryStore(user_id="alice_123")

# After conversation turn: extract and store important facts
memory.remember("User is a Python developer building a REST API", category="profile", importance=0.9)
memory.remember("User prefers short, direct answers", category="preference", importance=0.8)
memory.remember("User is using FastAPI with PostgreSQL", category="tech_stack", importance=0.9)

# At start of next session: retrieve relevant context
relevant_memories = memory.recall("Help me with my API project")
print("Recalled memories:")
for m in relevant_memories:
    print(f"  [{m['category']}] {m['fact']}")
```

### 2.3 Episodic Memory: Records of What Happened

Episodic memory stores the *events* of past interactions, not just facts:

```python
import json
from dataclasses import dataclass, asdict
from datetime import datetime

@dataclass
class Episode:
    """A record of a past agent interaction."""
    episode_id:  str
    user_id:     str
    timestamp:   str
    task:        str
    outcome:     str        # "success" | "failure" | "partial"
    key_actions: list[str]  # what the agent did
    result:      str        # the final output
    feedback:    str = ""   # user feedback if available

class EpisodicMemoryStore:
    """
    Stores past episodes and retrieves similar ones for in-context learning.
    "What did I do in similar situations before?"
    """
    
    def __init__(self, db_path: str = "episodes.json"):
        self.db_path  = db_path
        self.episodes = self._load()
        self.embeddings = {}   # episode_id → embedding (cached)
    
    def _load(self) -> list[Episode]:
        try:
            with open(self.db_path) as f:
                return [Episode(**ep) for ep in json.load(f)]
        except FileNotFoundError:
            return []
    
    def _save(self):
        with open(self.db_path, "w") as f:
            json.dump([asdict(ep) for ep in self.episodes], f, indent=2)
    
    def store(self, episode: Episode):
        self.episodes.append(episode)
        self._save()
    
    def find_similar(self, task: str, k: int = 3) -> list[Episode]:
        """Find episodes where the agent handled similar tasks."""
        import numpy as np
        
        task_vec = embed.encode([task], normalize_embeddings=True)[0]
        
        scored = []
        for ep in self.episodes:
            if ep.episode_id not in self.embeddings:
                self.embeddings[ep.episode_id] = embed.encode([ep.task], normalize_embeddings=True)[0]
            
            sim = float(np.dot(task_vec, self.embeddings[ep.episode_id]))
            scored.append((sim, ep))
        
        scored.sort(key=lambda x: -x[0])
        return [ep for _, ep in scored[:k]]
    
    def build_few_shot_context(self, task: str) -> str:
        """Build a few-shot example block from similar past episodes."""
        similar = self.find_similar(task, k=2)
        if not similar:
            return ""
        
        examples = []
        for ep in similar:
            if ep.outcome == "success":
                examples.append(
                    f"Similar past task: {ep.task}\n"
                    f"Approach taken: {', '.join(ep.key_actions)}\n"
                    f"Result: {ep.result[:200]}\n"
                )
        
        if examples:
            return "Relevant past experience:\n" + "\n---\n".join(examples)
        return ""
```

### 2.4 Memory-Augmented Agent

Combining all memory types in a single agent:

```python
import anthropic

client = anthropic.Anthropic()

class MemoryAugmentedAgent:
    """
    An agent with short-term, long-term, and episodic memory.
    """
    
    def __init__(self, user_id: str, system_base: str):
        self.user_id  = user_id
        self.system_base    = system_base
        self.conversation   = ConversationMemory(system_base)
        self.semantic_mem   = SemanticMemoryStore(user_id=user_id)
        self.episodic_mem   = EpisodicMemoryStore()
    
    def chat(self, user_message: str) -> str:
        # 1. Retrieve relevant long-term memories
        long_term = self.semantic_mem.recall(user_message, k=3)
        long_term_text = ""
        if long_term:
            long_term_text = "\n".join(f"- {m['fact']}" for m in long_term)
        
        # 2. Retrieve relevant past episodes
        episode_context = self.episodic_mem.build_few_shot_context(user_message)
        
        # 3. Build augmented system prompt
        system = self.system_base
        if long_term_text:
            system += f"\n\nWhat you know about this user:\n{long_term_text}"
        if episode_context:
            system += f"\n\n{episode_context}"
        
        # 4. Add user message to conversation history
        self.conversation.add("user", user_message)
        
        # 5. Generate response
        response = client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=1024,
            system=system,
            messages=self.conversation.get_messages()
        )
        
        answer = response.content[0].text
        self.conversation.add("assistant", answer)
        
        # 6. Extract and store important new facts (async in production)
        self._extract_and_remember(user_message, answer)
        
        return answer
    
    def _extract_and_remember(self, user_msg: str, agent_response: str):
        """Use LLM to extract memorable facts from the exchange."""
        extraction = client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=256,
            messages=[{
                "role": "user",
                "content": f"""From this exchange, extract 0-3 facts worth remembering for future sessions.
Only extract facts that are: specific to this user, not obvious, and likely useful later.
Output as JSON: {{"facts": ["fact1", "fact2"]}}

User: {user_msg}
Agent: {agent_response}"""
            }]
        )
        
        import json
        try:
            data = json.loads(extraction.content[0].text)
            for fact in data.get("facts", []):
                self.semantic_mem.remember(fact)
        except json.JSONDecodeError:
            pass
```

## 3. Real-World Context

### The Episodic Memory Paper (2025)

A 2025 position paper, "Episodic Memory is the Missing Piece for Long-Term LLM Agents,"
argued that LLM agents need five key properties for usable episodic memory:
- **Long-term storage:** Persistence beyond the session
- **Single-shot learning:** Capture information from one exposure without gradient updates
- **Instance-specific:** Details unique to each occurrence
- **Contextual:** Who, when, where, why — bound to the memory content
- **Explicit reasoning:** Ability to reflect on memory content

Current embedding-based episodic memory satisfies most of these. Fine-tuning doesn't
(it requires many examples). RAG over conversation logs satisfies some.

### Production Memory Architectures (2026)

| Architecture | Memory Types | Best For |
|---|---|---|
| Context window only | Short-term | Single-session assistants |
| Sliding window + summarization | Short-term | Long conversations |
| Vector store for user profiles | Long-term | Personalized assistants |
| Vector store for episodes | Episodic | Complex task history |
| Full stack (all three) | All | Enterprise agents, long-running workflows |

## 4. Trade-offs

### Memory Storage Approaches

| Approach | Retrieval | Storage | Update | Best For |
|---|---|---|---|---|
| Key-value store (Redis) | O(1) by key | Low | Easy | Explicit facts, user preferences |
| Relational DB | Query-based | Medium | Easy | Structured facts, searchable history |
| Vector DB | Semantic ANN | Medium | Easy (upsert) | Fuzzy retrieval, similar-context lookup |
| Full context | None (in window) | Temporary | N/A | Short-term; no infrastructure needed |

### The Memory-Privacy Trade-off

Storing user interactions creates privacy obligations:
- Long-term memory stores personal information
- Must comply with GDPR, CCPA, and similar regulations
- Users should be able to request deletion ("forget me")
- Store only what's necessary for the agent's function

Design memory stores with a `forget_user(user_id)` operation from day one.

## 5. When To Use vs. When To Avoid

### Long-Term Memory
✅ Use when: Users return across sessions and personalization matters
✅ Use when: Factual user context is needed (tech stack, preferences, history)
❌ Skip when: Single-use or anonymous interactions

### Episodic Memory
✅ Use when: The agent performs complex multi-step tasks and should improve over time
✅ Use when: You want the agent to learn from similar past situations
❌ Skip when: Tasks are all unique and past episodes don't generalize

⚠️ Common mistakes:
- Storing everything in long-term memory — noise drowns out signal; be selective
- Not summarizing context — memory systems without compression fill up and slow down
- Ignoring privacy/compliance requirements — user data has legal obligations
- Not providing a way to clear/update memories — incorrect facts persist otherwise

## 6. Senior AI Engineer Perspective

### Memory Is Not Intelligence

A common misconception: giving an agent more memory makes it smarter. Memory makes it
*more consistent* and *more personalized*, not more capable.

An agent that remembers your coding preferences is more useful. An agent that remembers
but still reasons poorly isn't improved by the memory.

Build capability first. Add memory when the agent is capable and you need persistence.

### The Memory Retrieval Problem

Long-term memory only helps if the right memories are retrieved at the right time. The
challenge: "relevant" is context-dependent and hard to capture perfectly.

Strategies that work:
- **Proactive injection:** Always inject user profile + preferences at session start
- **Reactive retrieval:** Query memory with the current user message as search query
- **Category filtering:** Retrieve memories by category (preferences, tech stack, history)
- **Recency weighting:** More recent memories score higher by default

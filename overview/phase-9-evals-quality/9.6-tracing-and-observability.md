## 1. What Is It? (Theory)

### Why LLM Observability Is Different From Standard APM

Traditional application monitoring tells you: CPU, memory, latency, error rate. When something
breaks, you check the logs and stack traces.

LLM applications have a fundamentally different failure mode profile:
- The API returns 200 OK. There's no exception.
- The output looks plausible. There's no obvious error.
- But the response quality degraded — slowly, silently.
- Standard APM tells you nothing about this.

You need **LLM-specific observability** that captures:
- What prompt was sent, in full
- What tokens were generated, step by step (for agents)
- What tools were called, with what inputs and outputs
- What each call cost
- How quality evolved over time

This is the difference between knowing your service is *running* and knowing your AI is *working*.

### The Three Layers

```
Layer 1: INFRASTRUCTURE OBSERVABILITY (standard APM)
  - Datadog, New Relic, Honeycomb
  - Host metrics, request latency, error rates
  - Answers: "Is the service up? Are requests failing?"

Layer 2: LLM OBSERVABILITY (LangSmith, Langfuse, Helicone)
  - Per-request tracing: prompt + response + tool calls
  - Cost tracking, latency per model call
  - Quality scoring integration
  - Answers: "Are the AI responses good? What changed?"

Layer 3: BUSINESS OBSERVABILITY (custom)
  - Task completion rates, user satisfaction signals
  - Downstream impact metrics
  - Answers: "Is the AI feature achieving its goal?"
```

All three layers are needed in production. Most teams have Layer 1 but skip Layer 2.

## 2. How It Actually Works — Deep Mechanics

### 2.1 What a Good Trace Looks Like

A trace for a RAG pipeline:

```
Trace: user_session_abc123
├── [150ms] RAG Query: "What is our return policy?"
│   ├── [12ms] embed_query
│   │   └── Model: voyage-3-large | Tokens: 8 | Cost: $0.000001
│   ├── [5ms] vector_search
│   │   └── Collection: knowledge_base | Top-K: 5 | Scores: [0.89, 0.82, ...]
│   ├── [120ms] generate_answer
│   │   ├── Model: claude-sonnet-4-6
│   │   ├── Input tokens: 847 | Output tokens: 124
│   │   ├── Cost: $0.0044
│   │   ├── System prompt: "You are a customer support assistant..."
│   │   └── Response: "Our return policy allows returns within 30 days..."
│   └── [2ms] citation_check
│       └── Citations found: 2 | All grounded: true
└── Total cost: $0.0044 | Total latency: 150ms
```

For an agent:

```
Trace: agent_session_xyz789
├── [2100ms] Research Agent Task
│   ├── [200ms] Iteration 1: Think + search_web("machine learning basics")
│   │   └── Tool result: {"results": [...]}
│   ├── [180ms] Iteration 2: Think + search_web("neural network history")
│   │   └── Tool result: {"results": [...]}
│   ├── [900ms] Iteration 3: Think + synthesize_response
│   │   └── Model: claude-sonnet-4-6 | Tokens: 2847 | Cost: $0.022
│   └── Total iterations: 3 | Total cost: $0.028
```

### 2.2 Langfuse (Open Source — Recommended)

```python
# pip install langfuse anthropic
from langfuse import Langfuse
from langfuse.decorators import observe, langfuse_context
import anthropic

langfuse = Langfuse(
    public_key=os.environ["LANGFUSE_PUBLIC_KEY"],
    secret_key=os.environ["LANGFUSE_SECRET_KEY"],
    host="https://cloud.langfuse.com"  # or self-hosted
)

# Method 1: Decorator-based (simplest)
@observe(name="rag_query")
def rag_with_tracing(question: str, user_id: str) -> str:
    """Decorated function — Langfuse automatically traces inputs/outputs."""
    
    # Add metadata to the current trace
    langfuse_context.update_current_trace(
        user_id=user_id,
        metadata={"feature": "support_chat"}
    )
    
    # Retrieval span
    with langfuse_context.observe(name="retrieval"):
        chunks = retrieve(question)
        langfuse_context.update_current_observation(
            output={"chunks_count": len(chunks), "top_score": chunks[0]["score"] if chunks else 0}
        )
    
    # Generation span
    client = anthropic.Anthropic()
    with langfuse_context.observe(name="generation", model="claude-sonnet-4-6"):
        response = client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=512,
            system="Answer from provided context.",
            messages=[{"role": "user", "content": f"Context: {build_context(chunks)}\n\nQuestion: {question}"}]
        )
        answer = response.content[0].text
        
        # Track usage for cost calculation
        langfuse_context.update_current_observation(
            usage={
                "input": response.usage.input_tokens,
                "output": response.usage.output_tokens,
            }
        )
    
    return answer

# Method 2: Manual SDK (more control)
def rag_manual_trace(question: str, user_id: str) -> str:
    trace = langfuse.trace(
        name="rag_query",
        user_id=user_id,
        metadata={"feature": "support_chat"},
        input={"question": question},
    )
    
    # Retrieval span
    retrieval_span = trace.span(name="retrieval")
    chunks = retrieve(question)
    retrieval_span.end(output={"n_chunks": len(chunks)})
    
    # Generation generation
    gen_span = trace.generation(
        name="generation",
        model="claude-sonnet-4-6",
        input=[{"role": "user", "content": question}],
        model_parameters={"max_tokens": 512}
    )
    
    client = anthropic.Anthropic()
    response = client.messages.create(
        model="claude-sonnet-4-6", max_tokens=512,
        messages=[{"role": "user", "content": question}]
    )
    answer = response.content[0].text
    
    gen_span.end(
        output=answer,
        usage={"input": response.usage.input_tokens, "output": response.usage.output_tokens}
    )
    
    # Add quality score (from LLM judge or user feedback)
    trace.score(name="helpfulness", value=0.9, comment="Correctly answered with citation")
    
    trace.update(output={"answer": answer})
    return answer
```

### 2.3 Helicone (Simplest Setup — Proxy-Based)

Helicone intercepts API calls via a proxy — no SDK changes needed, just change the base URL:

```python
import anthropic

# Standard client
standard_client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

# Helicone client (just change the base URL)
helicone_client = anthropic.Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"],
    base_url="https://anthropic.helicone.ai",
    default_headers={
        "Helicone-Auth": f"Bearer {os.environ['HELICONE_API_KEY']}",
        "Helicone-User-Id":    "user_123",
        "Helicone-Property-Feature": "support_chat",
        "Helicone-Property-Env":     "production",
        # All API calls are automatically logged with cost, latency, tokens
    }
)

# Usage: identical to standard client
response = helicone_client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=512,
    messages=[{"role": "user", "content": "What is our return policy?"}]
)
# Helicone automatically records: prompt, response, cost, latency, user_id
```

**Helicone's advantage:** Zero code changes to existing LLM calls. Add the header, get observability.
**Helicone's limitation:** API-call-level tracing only. For agent step-by-step visibility, you need
LangSmith or Langfuse.

### 2.4 LangSmith (Best for LangChain/LangGraph)

```python
import os
from langsmith import traceable
from langchain_anthropic import ChatAnthropic
from langchain_core.messages import HumanMessage

os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"]    = os.environ["LANGSMITH_API_KEY"]
os.environ["LANGCHAIN_PROJECT"]    = "my-rag-project"

# With @traceable decorator
@traceable(name="rag_query", tags=["rag", "production"])
def rag_with_langsmith(question: str) -> str:
    # All LangChain operations inside are automatically traced
    llm = ChatAnthropic(model="claude-sonnet-4-6")
    
    # LangSmith captures: input, output, latency, cost, model params
    response = llm.invoke([HumanMessage(content=question)])
    return response.content

# For LangGraph agents: every node is automatically traced as a separate span
# with state diffs, tool calls, and iteration counts
```

### 2.5 Production Monitoring Dashboard Setup

```python
import anthropic
from langfuse import Langfuse
from datetime import datetime, timedelta

langfuse = Langfuse()

def get_quality_metrics(
    lookback_days: int = 7,
    feature: str | None = None,
) -> dict:
    """Aggregate quality metrics from Langfuse for a dashboard."""
    
    # Query traces from last N days
    traces = langfuse.client.trace.list(
        from_timestamp=datetime.utcnow() - timedelta(days=lookback_days),
        metadata={"feature": feature} if feature else None,
        limit=1000
    )
    
    # Compute key metrics
    total_cost = sum(t.total_cost for t in traces.data if t.total_cost)
    avg_latency = sum(t.latency for t in traces.data if t.latency) / max(len(traces.data), 1)
    
    scores = langfuse.client.score.list(
        from_timestamp=datetime.utcnow() - timedelta(days=lookback_days),
        name="helpfulness"
    )
    avg_quality = sum(s.value for s in scores.data) / max(len(scores.data), 1)
    
    return {
        "period":         f"Last {lookback_days} days",
        "total_requests": len(traces.data),
        "total_cost_usd": round(total_cost, 2),
        "avg_latency_ms": round(avg_latency, 0),
        "avg_quality":    round(avg_quality, 2),
        "quality_samples": len(scores.data),
    }
```

## 3. Real-World Context

### The 2026 Production Stack

Teams that operate production AI systems at scale typically run:

```
Infrastructure layer:
  Datadog / New Relic → host metrics, request rates, error rates

LLM observability layer:
  Langfuse (open source) → traces, costs, evals, prompt versions
  OR LangSmith → for LangChain/LangGraph-heavy stacks
  OR Helicone → simplest setup for teams without custom agents

Alerting:
  PagerDuty / Slack → fires when cost spike, quality drop, error rate spike
```

### Cost Monitoring: The Number One Alert

The most commonly triggered alert in production AI systems is cost. A runaway agent loop,
a suddenly popular feature, or an unexpected usage pattern can cause costs to spike 10–100×
in hours. Standard cloud billing alerts have hours of delay. Langfuse/Helicone cost alerts
can fire in real-time.

```python
# Langfuse alert configuration (conceptual)
langfuse.create_alert(
    name="cost_spike",
    condition="rolling_hourly_cost > 5.00",   # Alert if >$5/hour
    action="slack_webhook",
    channel="#ai-ops"
)

langfuse.create_alert(
    name="quality_drop",
    condition="rolling_avg_score(window=1h) < 3.5",
    action="pagerduty"
)
```

## 4. Providers, Tools & Ecosystem

### Observability Platform Comparison (2026)

| Platform | Type | Tracing Depth | Eval Integration | Self-Host | Best For |
|---|---|---|---|---|---|
| **Langfuse** | Open source / cloud | Deep (agent steps) | Yes (custom scores) | Yes (Postgres + ClickHouse) | Teams wanting full control |
| **LangSmith** | Commercial | Deepest for LangChain | Yes (LangChain evals) | No | LangChain/LangGraph users |
| **Helicone** | Commercial | API-call level | Limited | Yes | Simplest setup; proxy-based |
| **Arize Phoenix** | Open source / cloud | Deep | Strong (ML evals) | Yes | ML-ops teams |
| **Braintrust** | Commercial | Deep | Strong (eval-first) | No | Teams prioritizing evals |
| **Datadog LLM Obs** | Commercial | Medium | Limited | No | Existing Datadog customers |

### Getting Started Recommendation

1. **Smallest team / prototype stage:** Helicone — change one URL, get cost + latency tracking
2. **Growing team / production feature:** Langfuse — open source, self-hostable, full-featured
3. **LangChain shop:** LangSmith — deepest integration
4. **ML-ops team with existing MLflow:** Arize Phoenix — integrates with MLflow

## 5. What to Trace at Each Level

```python
# EVERY LLM call should capture:
trace_fields = {
    "request_id":     "uuid per request",
    "user_id":        "anonymized user identifier",
    "feature_name":   "which product feature triggered this",
    "model":          "exact model version used",
    "input_tokens":   "from response.usage",
    "output_tokens":  "from response.usage",
    "cost_usd":       "computed from tokens + model pricing",
    "latency_ms":     "time from request to first token (TTFT) + total",
    "environment":    "production | staging | dev",
}

# RAG pipeline should also capture:
rag_trace_fields = {
    "query":              "original user question",
    "retrieval_latency":  "time for vector search",
    "n_chunks_retrieved": "how many chunks were fetched",
    "top_chunk_score":    "similarity score of best chunk",
    "context_tokens":     "tokens in the retrieved context",
    "citation_verified":  "bool: did citations check out?",
}

# Agent loops should also capture:
agent_trace_fields = {
    "n_iterations":   "how many tool-call loops",
    "tools_called":   "list of tool names + args",
    "loop_detected":  "bool: did agent loop on same tool?",
    "final_status":   "completed | max_iterations | error",
}
```

## 1. What Is It? (Theory)

### The Non-Determinism Challenge

In traditional software, regression testing is straightforward:
```python
assert my_function(input) == expected_output  # deterministic: always same result
```

In AI systems:
```python
response1 = llm("What is the capital of France?")  # "Paris"
response2 = llm("What is the capital of France?")  # "The capital of France is Paris."
response3 = llm("What is the capital of France?")  # "Paris, which has been France's capital since..."
# All three are correct. None match each other exactly.
```

You cannot use exact-match assertions. You need **statistical tests** and **semantic checks**
that tolerate valid variation while detecting actual regressions.

### What a Regression Looks Like

```
Week 1:  Average helpfulness score: 4.2/5
Week 2:  Average helpfulness score: 4.1/5  ← noise, within tolerance
Week 3:  Average helpfulness score: 3.7/5  ← REGRESSION (>0.3 drop)

Cause: Provider updated the model silently.
Without regression testing: users complain 2 weeks later.
With regression testing: alert fires 1 hour after the model update.
```

## 2. How It Actually Works — Deep Mechanics

### 2.1 The Statistical Approach

Instead of checking exact outputs, check whether the **distribution of quality scores**
has shifted significantly:

```python
import numpy as np
from scipy import stats
from dataclasses import dataclass
from typing import Optional

@dataclass
class EvalRun:
    run_id:        str
    model:         str
    prompt_version: str
    timestamp:     str
    scores:        list[float]
    
    @property
    def mean(self) -> float:
        return np.mean(self.scores)
    
    @property
    def p10(self) -> float:
        return np.percentile(self.scores, 10)   # worst 10%

def detect_regression(
    baseline: EvalRun,
    current:  EvalRun,
    mean_drop_threshold: float = 0.2,   # alert if mean drops by this much
    p10_drop_threshold:  float = 0.3,   # alert if worst 10% drops by this much
    significance_level:  float = 0.05,  # p-value threshold for statistical test
) -> dict:
    """
    Detect statistically significant regression between two eval runs.
    Uses both absolute thresholds and statistical significance.
    """
    mean_delta = current.mean - baseline.mean
    p10_delta  = current.p10  - baseline.p10
    
    # Mann-Whitney U test: are the distributions significantly different?
    stat, p_value = stats.mannwhitneyu(baseline.scores, current.scores, alternative="two-sided")
    
    is_regression = (
        mean_delta < -mean_drop_threshold or
        p10_delta  < -p10_drop_threshold
    ) and p_value < significance_level
    
    return {
        "regression_detected": is_regression,
        "mean_baseline":  round(baseline.mean, 3),
        "mean_current":   round(current.mean, 3),
        "mean_delta":     round(mean_delta, 3),
        "p10_baseline":   round(baseline.p10, 3),
        "p10_current":    round(current.p10, 3),
        "p10_delta":      round(p10_delta, 3),
        "p_value":        round(p_value, 4),
        "statistically_significant": p_value < significance_level,
        "message": f"{'⚠️  REGRESSION' if is_regression else '✓ No regression'}: "
                   f"mean {baseline.mean:.2f} → {current.mean:.2f} ({mean_delta:+.2f})"
    }
```

### 2.2 The Eval CI/CD Pipeline

```python
import subprocess
from pathlib import Path
import json
from datetime import datetime

def run_regression_suite(
    eval_cases: list[dict],
    system_prompt: str,
    model: str,
    judge_fn,
    baseline_path: str,
    pass_threshold: float = 0.75,     # minimum pass rate to not block
    regression_delta: float = 0.1,    # maximum acceptable mean score drop
) -> dict:
    """
    Full regression test suite.
    Returns pass/fail verdict for CI/CD gate.
    """
    import anthropic
    client = anthropic.Anthropic()
    
    results = []
    for case in eval_cases:
        # Generate output
        response = client.messages.create(
            model=model, max_tokens=512,
            system=system_prompt,
            messages=[{"role": "user", "content": case["input"]}],
            temperature=0   # deterministic for regression testing
        )
        output = response.content[0].text
        
        # Score with deterministic check
        deterministic_pass = case.get("expected_keyword", "").lower() in output.lower()
        
        # Score with judge (for qualitative criteria)
        judge_result = judge_fn(case["input"], output)
        judge_score  = judge_result.get("overall", 0)
        
        results.append({
            "case_id":    case.get("case_id", ""),
            "passed_det": deterministic_pass,
            "judge_score": judge_score,
            "output_preview": output[:100]
        })
    
    # Compute summary metrics
    det_pass_rate = sum(r["passed_det"] for r in results) / len(results)
    mean_score    = np.mean([r["judge_score"] for r in results])
    
    # Load baseline if it exists
    baseline_file = Path(baseline_path)
    regression_info = None
    
    if baseline_file.exists():
        with open(baseline_file) as f:
            baseline_data = json.load(f)
        
        baseline_run = EvalRun(
            run_id="baseline", model=baseline_data["model"],
            prompt_version="baseline", timestamp=baseline_data["timestamp"],
            scores=baseline_data["scores"]
        )
        current_run = EvalRun(
            run_id="current", model=model,
            prompt_version="current", timestamp=datetime.utcnow().isoformat(),
            scores=[r["judge_score"] for r in results]
        )
        regression_info = detect_regression(baseline_run, current_run)
    
    # Determine CI/CD verdict
    verdict = "PASS"
    reasons = []
    
    if det_pass_rate < pass_threshold:
        verdict = "FAIL"
        reasons.append(f"Deterministic pass rate {det_pass_rate:.0%} < threshold {pass_threshold:.0%}")
    
    if regression_info and regression_info["regression_detected"]:
        verdict = "FAIL"
        reasons.append(regression_info["message"])
    
    # Save results as new baseline (if passing)
    if verdict == "PASS":
        save_data = {
            "model": model,
            "timestamp": datetime.utcnow().isoformat(),
            "scores": [r["judge_score"] for r in results],
            "mean_score": mean_score,
            "det_pass_rate": det_pass_rate
        }
        with open(baseline_path, "w") as f:
            json.dump(save_data, f, indent=2)
    
    return {
        "verdict":           verdict,
        "reasons":           reasons,
        "det_pass_rate":     det_pass_rate,
        "mean_judge_score":  mean_score,
        "n_cases":           len(results),
        "regression":        regression_info,
        "should_block":      verdict == "FAIL"
    }
```

### 2.3 GitHub Actions Integration

```yaml
# .github/workflows/llm-eval.yml
name: LLM Regression Tests

on:
  push:
    branches: [main]
    paths:
      - 'prompts/**'
      - 'src/ai/**'
      - '.github/workflows/llm-eval.yml'
  pull_request:
    paths:
      - 'prompts/**'
      - 'src/ai/**'
  schedule:
    # Run daily to catch silent model updates from provider
    - cron: '0 6 * * *'

jobs:
  eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with: {python-version: '3.11'}
      
      - name: Install dependencies
        run: pip install -r requirements-eval.txt
      
      - name: Run eval suite
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          python -m pytest evals/ -v \
            --eval-model=claude-sonnet-4-6 \
            --baseline=.eval_baselines/latest.json \
            --threshold=0.75 \
            --max-regression=0.1
      
      - name: Upload eval results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: eval-results-${{ github.run_id }}
          path: eval_results/
      
      - name: Comment on PR
        if: github.event_name == 'pull_request'
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const results = JSON.parse(fs.readFileSync('eval_results/summary.json'));
            const status = results.verdict === 'PASS' ? '✅' : '❌';
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## Eval Results ${status}\n` +
                    `**Verdict:** ${results.verdict}\n` +
                    `**Pass rate:** ${(results.det_pass_rate * 100).toFixed(0)}%\n` +
                    `**Mean quality score:** ${results.mean_judge_score.toFixed(2)}/5\n` +
                    (results.reasons.length ? `**Issues:**\n${results.reasons.map(r => `- ${r}`).join('\n')}` : '')
            });
```

### 2.4 pytest Integration

```python
# evals/test_regression.py
import pytest
import anthropic
import json
from pathlib import Path

client = anthropic.Anthropic()

def load_golden_dataset(path: str = "evals/golden_dataset.json") -> list[dict]:
    with open(path) as f:
        return json.load(f)["cases"]

GOLDEN_CASES = load_golden_dataset()

@pytest.mark.parametrize("case", [c for c in GOLDEN_CASES if c["category"] == "factual"])
def test_factual_accuracy(case: dict, eval_config):
    """Factual cases should answer correctly."""
    response = client.messages.create(
        model=eval_config.model,
        max_tokens=256,
        system=eval_config.system_prompt,
        messages=[{"role": "user", "content": case["input"]}],
        temperature=0
    )
    output = response.content[0].text
    
    # Check that expected keyword appears in output
    if case.get("expected_keyword"):
        assert case["expected_keyword"].lower() in output.lower(), (
            f"Expected '{case['expected_keyword']}' in output.\n"
            f"Got: {output[:200]}"
        )

@pytest.mark.parametrize("case", [c for c in GOLDEN_CASES if c.get("should_refuse")])
def test_refusal_cases(case: dict, eval_config):
    """Out-of-scope queries should be refused gracefully."""
    response = client.messages.create(
        model=eval_config.model,
        max_tokens=256,
        system=eval_config.system_prompt,
        messages=[{"role": "user", "content": case["input"]}],
        temperature=0
    )
    output = response.content[0].text
    
    # Should not answer the question; should redirect
    refusal_phrases = ["don't have", "out of scope", "can't help with", "not able to"]
    assert any(p in output.lower() for p in refusal_phrases), (
        f"Expected refusal for: '{case['input']}'\n"
        f"Got: {output[:200]}"
    )

class TestRegressionGate:
    """Regression tests that block CI if quality drops."""
    
    def test_overall_pass_rate_above_threshold(self, eval_results):
        """At least 80% of eval cases must pass."""
        assert eval_results["det_pass_rate"] >= 0.80, (
            f"Pass rate {eval_results['det_pass_rate']:.0%} below threshold 80%"
        )
    
    def test_no_regression_from_baseline(self, eval_results):
        """Score must not drop more than 0.1 from baseline."""
        if eval_results.get("regression"):
            assert not eval_results["regression"]["regression_detected"], (
                f"Regression detected: {eval_results['regression']['message']}"
            )
    
    def test_canary_cases_pass(self, eval_results):
        """All canary cases must pass — if these fail, something is fundamentally broken."""
        for case in eval_results.get("canary_results", []):
            assert case["passed"], (
                f"CANARY FAILED: '{case['input']}'\nGot: {case['output'][:200]}"
            )
```

## 3. Real-World Context

### The "Silent Model Update" Problem

LLM providers update their models without version bumping the model name. `claude-sonnet-4-6`
today is not necessarily the same model as `claude-sonnet-4-6` six months from now.

In 2025–2026, several teams experienced production degradations caused by silent model updates:
- Output style changed (shorter responses, different formatting)
- Specific edge cases started failing that previously worked
- Safety behavior changed (more or fewer refusals)

**Defense:** Run daily scheduled eval runs. If the model name is the same but scores dropped,
you know something changed on the provider side. You can then pin to a specific dated version
(e.g., `claude-sonnet-4-6-20251001`) until you've validated the new behavior.

### Statistical Power: How Many Cases Do You Need?

To detect a 5% quality drop with 80% power:
- Score variance σ = 0.8 (typical for LLM quality scores on 1–5 scale)
- Required n per group ≈ 200 cases

To detect a 10% drop:
- Required n ≈ 50 cases

**Practical rule:**
- 50 cases: can detect large (≥10%) regressions reliably
- 100 cases: can detect medium (≥7%) regressions
- 200+ cases: can detect subtle (≥5%) regressions

Most teams are fine with 50–100 cases if they set thresholds appropriately.

## 5. Trade-offs

### Temperature 0 vs Real Temperature for Regression Testing

**temperature=0 (recommended for CI/CD):**
- Deterministic; same result every run
- Tests the model's most-likely behavior
- May not cover the distribution of real outputs

**Real temperature:**
- More representative of production behavior
- Requires multiple samples per case (statistical)
- More expensive; slower

**Rule:** Use temperature=0 for CI/CD gates. Use real temperature for periodic deep analysis.

### How Often to Run

| Trigger | What to Run |
|---|---|
| Every PR | Fast deterministic checks + canary cases (10–20 cases, <1 min) |
| Every merge to main | Full golden dataset (50–100 cases, 5–10 min) |
| Daily (scheduled) | Full suite + provider update detection |
| Weekly | Deep analysis with LLM judge + statistical regression detection |

## 6. When To Use vs. When To Avoid

### Regression Testing
✅ Always: Any AI feature that has shipped and will be changed
✅ Always: As a deployment gate before releasing prompt or model changes
❌ Skip for: One-time batch jobs or throwaway scripts

⚠️ Common mistakes:
- Running regression tests only before major releases (too infrequent; you miss gradual drift)
- Using exact string matching for non-deterministic outputs (fails on valid variations)
- Setting thresholds too tight (many false positives; team learns to ignore alerts)
- Setting thresholds too loose (regressions slip through)
- Not updating the baseline when you intentionally change behavior (new features look like regressions)

## 7. Senior AI Engineer Perspective

### The Baseline Update Ceremony

When you intentionally change behavior (new prompt, new model, new feature), the regression
test suite will fail because the new behavior doesn't match the old baseline. This is expected.

The ceremony: review the failed cases, confirm the new behavior is intentional and correct,
update the baseline. This is not "bypassing" the regression test — it's the intended workflow.
The test protects against *unintended* regressions, not *intended* changes.

Document in your commit or PR: "Updated eval baseline — intentional behavior change for X feature."

### The Three Numbers Every Team Should Track

1. **Pass rate this week vs last week** — relative change catches regressions
2. **P10 score** (worst 10% of cases) — catches degradation in edge cases before mean drops
3. **Canary pass rate** — if any canary fails, everything stops

These three numbers, tracked weekly, catch 90% of AI quality regressions before users notice.
